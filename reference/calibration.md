# Reviewer calibration

Good and bad examples per review dimension. Use these to calibrate judgment — especially for edge cases and borderline copy.

---

## Glossary

✓ "Back to Workspace" — correct term, correct context (Sticklight start screen)
✓ "Manage team access from your Workspace settings" — correct term, correct context (Pink container)
✓ "Use Skills to teach the AI how to handle repeatable tasks" — correct; Skills = workflow instructions
✓ "Start from this template and make it your own" — correct; template = visual starting point
✓ "Connect your GitHub account" — correct; account = GitHub entity, not Pink workspace
✓ "Open the Chat panel to continue working with Sticklight" — correct term

❌ "Back to homepage" — use Workspace
❌ "Generate your project" — use Create as the primary CTA
❌ "Upload a template to tell Sticklight how to handle this workflow" — use Skills, not template
❌ "Open the AI panel" — use Chat panel
❌ "Enter Research mode" — use Plan mode
❌ "Use your GitHub workspace" — use GitHub account or GitHub organization

Do NOT flag:
- "brand assets" — not an avoided term
- "template" when the copy is about an actual template
- "account" or "organization" in a GitHub context

---

## Voice & Tone

✓ "You lead. Sticklight follows." — creator agency, collaborative framing
✓ "Select an element to start editing" — functional tone during active building
✓ "What would you like to build today?" — collaborative tone during exploration
✓ "The integration couldn't connect. Check your API key and try again." — diagnostic tone in error state
✓ "Your site is live." — milestone moment (launching), short and direct
✓ "Deleting this project is permanent and cannot be undone." — systemic tone for destructive action

❌ "We're reimagining what's possible for the future of the web" — inflated, no tangible value
❌ "Sticklight can do what designers used to" — removes creator agency
❌ "Amazing! You just created a project! 🎉" — celebratory outside a milestone moment
❌ "Oops! Something went wrong. We're on it! 😅" — wrong tone for error state + emoji in alert
❌ "Chat with the AI bot to refine your project" — Sticklight is not a bot
❌ "Open Sticklight's Chat panel" — restructure to avoid possessive: "the Chat panel in Sticklight"
❌ "Built for users who want to ship fast" — use "creators", not "users"

Milestone moments where celebratory copy is appropriate:
- Joining Sticklight
- Launching a website
- Adding a Skill

## Grammar & Style

✓ "Site settings" — sentence case ✓
✓ "Layout generated" — active voice ✓
✓ "3 issues found across 2 screens" — numerals ✓
✓ "You'll need to reconnect your GitHub account" — contraction ✓
✓ "Select an element to start editing" — single sentence, no period ✓
✓ "Deleting this project is permanent. This action cannot be undone." — two sentences, both with periods ✓

❌ "Site Settings" — use sentence case
❌ "The layout has been generated" — use active voice: "Layout generated"
❌ "Three issues found" — use numerals: "3 issues found"
❌ "You will need to reconnect" — use contraction: "You'll need to reconnect"
❌ "Select an element to start editing." — remove period from single sentence
❌ "Just add a widget — it's super easy!" — avoid "just" and "easy"

---

## UX Writing

✓ "No projects yet. Describe what you want to build and Sticklight will create it." — empty state as invitation
✓ "Couldn't publish. Check your domain settings and try again." — error with cause + next step
✓ "Project name" — placeholder as short label ✓
✓ "Learn about Skills" — descriptive link text ✓
✓ "Your project is ready." — success state as short sentence ✓
✓ "Deleting this project is permanent. This action cannot be undone." — destructive action with consequence

❌ "No projects found." — empty state as absence
❌ "An error occurred." — error with no cause or next step
❌ "Enter the name of your project here..." — placeholder as full sentence
❌ "Learn more" — vague link text; describe the destination
❌ "Created." — bare verb for success state; use a short sentence
❌ "Are you sure you want to delete?" — doesn't explain the consequence

Conversational design (Chat panel responses):
✓ "Got it — I'll redesign the hero section with your brand colors and add a contact form below." — echoes complex instruction, clear plan
✓ "I've added a hero section with a standard layout. Let me know if you'd like to adjust it." — acts on under-specified prompt, invites refinement
❌ "Before I start, can you tell me: What layout? What image? What headline?" — unnecessary questions that halt momentum
❌ "I have generated the layout for your project." — passive, formal; use "Layout generated" or "Here's your layout"

---

## Inclusive Language

✓ "Invite your team to collaborate" — gender-neutral ✓
✓ "Run a confidence check before publishing" — replaces "sanity check" ✓
✓ "Add this domain to the allowlist" — replaces "whitelist" ✓
✓ "They can update their settings at any time" — gender-neutral singular pronoun ✓

❌ "Invite your guys to collaborate" — use "team" or "creators"
❌ "Run a sanity check before publishing" — use "confidence check"
❌ "Add this domain to the whitelist" — use "allowlist"
❌ "Each creator can update his settings" — use "their"
❌ "That's a crazy amount of flexibility" — use "remarkable" or cut the intensifier

## Writing quality (silent — apply in improved_text only, do not flag)

These patterns appear in copy that's technically correct but sounds generated or formal. Fix them silently in the rewrite.

| Formal | Natural |
|---|---|
| "Provide your brand assets" | "Add your brand assets" |
| "Configure the build" | "Set up the build" |
| "Utilize the Skills Library" | "Use the Skills Library" |
| "Ensure your API key is valid" | "Make sure your API key is valid" |
| "Set this template as your project's baseline" | "Start from this template and make it your own" |
| "Provide the relevant requirements to generate a build tailored to your specific context" | "Add your requirements and Sticklight will tailor the project to your context" |
| "In order to create a project" | "To create a project" |
| "The project has been successfully created" | "Your project is ready" |
| "Feel free to add your brand assets" | "Add your brand assets" |
| "This will allow you to customize the layout" | "Customize the layout" |
| "The project could potentially be improved by adding a brand kit" | "Add a brand kit to improve the project" |
| "Please enter your API key" | "Enter your API key" |
| "Don't worry, you can always change this later" | "You can update this any time" |
