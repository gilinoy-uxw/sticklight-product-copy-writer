# Term clashes

Some terms mean different things in Sticklight and Pink. Always apply the definition for the declared PRODUCT. Never cross-apply.

## Workspace

| Product | Meaning |
|---|---|
| Sticklight | The creation start screen where users begin a new project by entering a prompt |
| Pink | The account-level container where users manage projects, environments, team settings, and billing |

❌ "Open the workspace to start prompting" — wrong in Pink (that's not what Workspace means there)
❌ "Manage billing from the Workspace" — wrong in Sticklight (that's not what Workspace means there)

## Project

| Product | Meaning |
|---|---|
| Sticklight | A saved digital product (website, app, dashboard, store) created with the builder |
| Pink | The managed unit being converted, hosted, configured, and deployed |

Note: the term overlaps intentionally — a Sticklight project can become a Pink project. Context determines which definition applies.

## Build

| Product | Meaning |
|---|---|
| Sticklight | Flexible supporting copy alongside Create — "Sticklight builds your project" |
| Pink | A specific technical pipeline step: the phase where Pink prepares the repository for deployment |

❌ "Click Build to start" — wrong in Sticklight (Create is the primary CTA)
❌ "Users build their projects using Sticklight" — acceptable in Sticklight copy as supporting verb
✓ "Build" — correct as a step name in Pink

## Account / Organization

| Context | Correct term |
|---|---|
| Referring to a GitHub account | "GitHub account" or "account" |
| Referring to a GitHub organization | "GitHub organization" or "organization" |
| Referring to the Pink team container | "Workspace" — NOT "account" or "organization" |

❌ "Choose a workspace that owns this repository" — wrong (Workspace is Pink's container, not GitHub's)
✓ "Select the account or organization that owns this repository" — correct (GitHub context)

## Templates vs Skills

These are not a product clash — they're distinct concepts within Sticklight itself.

| Term | What it is |
|---|---|
| Templates | Visual starting points — pre-built websites creators start from |
| Skills | Reusable workflow instructions that tell Sticklight how to build |

❌ Flagging "template" when the copy is genuinely about a template — do NOT do this
✓ Flag "template" only when it's used instead of "Skills" to describe reusable workflow instructions
